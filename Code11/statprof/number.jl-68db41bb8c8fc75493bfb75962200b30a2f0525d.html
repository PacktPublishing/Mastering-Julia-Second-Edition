311 """
312 one(x)
313 one(T::type)
314
315 Return a multiplicative identity for `x`: a value such that
316 `one(x)*x == x*one(x) == x`. Alternatively `one(T)` can
317 take a type `T`, in which case `one` returns a multiplicative
318 identity for any `x` of type `T`.
319
320 If possible, `one(x)` returns a value of the same type as `x`,
321 and `one(T)` returns a value of type `T`. However, this may
322 not be the case for types representing dimensionful quantities
323 (e.g. time in days), since the multiplicative
324 identity must be dimensionless. In that case, `one(x)`
325 should return an identity value of the same precision
326 (and shape, for matrices) as `x`.
327
328 If you want a quantity that is of the same type as `x`, or of type `T`,
329 even if `x` is dimensionful, use [`oneunit`](@ref) instead.
330
331 See also the [`identity`](@ref) function,
332 and `I` in [`LinearAlgebra`](@ref man-linalg) for the identity matrix.
333
334 # Examples
335 ```jldoctest
336 julia> one(3.7)
337 1.0
338
339 julia> one(Int)
340 1
341
342 julia> import Dates; one(Dates.Day(1))
343 1
344 ```
